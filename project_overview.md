# Project Overview ## **Autonomous Ticket Triage and Routing Using Retrieval-Augmented Generative AI** ### 1️⃣ What the Project Is In large software companies, engineers use ticketing systems like **Jira**, **ServiceNow**, or **GitHub Issues** to report bugs, incidents, and feature requests. When a ticket is created, someone must manually: * read the issue description * understand the problem * decide its **severity** * assign it to the correct **team or developer** This manual process is called **ticket triage**, and it takes time. Our project builds an **AI agent that automatically performs this triage process**. So the system will: * read the ticket * understand the problem * classify the issue * assign it to the correct team * suggest initial debugging steps. --- # 2️⃣ What the Input and Output Are ### Input A **ticket from GitHub Issues** containing: * title * description * labels * comments. Example:
Title: VSCode crashes when opening large file
Description: The editor freezes when opening files above 200MB.
### Output The AI system will produce: * Issue type (bug / feature request / improvement) * Severity level * Assigned team or developer * Suggested debugging steps --- # 3️⃣ How We Are Going to Build It The system will work in **four stages**. ### Step 1 — Ticket Understanding We use a **Large Language Model (LLM)** to read the ticket text and understand the problem. The model analyzes: * the title * the description * error messages or logs. --- ### Step 2 — Classification and Routing The system predicts: * ticket category * priority * responsible team. For example: | Ticket | Assigned Team | | ----------------- | ------------------- | | UI bug | Frontend team | | Database error | Backend team | | Performance issue | Infrastructure team | --- ### Step 3 — Retrieval-Augmented Generation (RAG) Instead of relying only on the AI model, we retrieve **similar past tickets** from the dataset. This is called **Retrieval‑Augmented Generation**. The system: 1. searches past resolved tickets 2. retrieves relevant ones 3. gives them to the AI model as context. This helps the AI give **better debugging suggestions**. --- ### Step 4 — Response Generation Finally the AI agent generates: * routing decision * explanation * debugging suggestions. Example output:
Category: Bug
Priority: Medium
Assigned Team: File System Team

Suggested Debugging Steps:
1. Check file buffer allocation
2. Inspect file loading module
3. Review recent commits related to file handling
--- # 4️⃣ Dataset Used We use public issue data from the **Microsoft** repository: **Visual Studio Code GitHub Issues** Dataset link: [https://github.com/microsoft/vscode/issues](https://github.com/microsoft/vscode/issues) This dataset contains: * thousands of real software issue reports * labels and comments * debugging discussions. These are used as: * **training data** * **RAG knowledge base**. --- # 5️⃣ Why This Project Is Useful The system helps organizations: * reduce manual ticket triage time * improve routing accuracy * help engineers start debugging faster. Instead of spending time deciding **who should handle the ticket**, engineers can directly start **fixing the problem**. --- # 6️⃣ One-Line Summary (Best Closing Line for Review) You can end with this sentence: > “Our project develops an AI-based autonomous agent that reads software issue tickets, classifies their severity, routes them to the appropriate team, and generates debugging suggestions using large language models and retrieval-augmented generation.”
